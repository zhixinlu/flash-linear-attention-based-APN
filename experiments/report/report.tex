\begin{abstract}
We benchmark three sequence model architectures on the sequential CIFAR-10 task:
the \textbf{Associative Plasticity Network (APN)}, mapped onto the gated delta-rule Triton kernel;
\textbf{DeltaNet}, a delta-rule linear attention model; and a standard \textbf{softmax Transformer} with FlashAttention-2.
All experiments use 10 layers, run for 200 epochs on a single NVIDIA H200 GPU, and share the same outer architecture
(input projection $\to$ residual layer stack $\to$ classifier).
We compare accuracy, training speed, GPU memory, and parameter efficiency across both
width-matched and parameter-matched configurations.
\end{abstract}

% ============================================================================
\section{Introduction}

The Associative Plasticity Network (APN) is a biologically-inspired recurrent model that maintains a Hebbian plastic memory matrix $M_t$, updated at every time-step.
By mapping the APN recurrence onto the \emph{gated delta-rule} chunk kernel from the Flash Linear Attention (FLA) library, we achieve efficient GPU training with $O(TD)$ complexity instead of the $O(T^2 D)$ cost of standard softmax attention.

This report summarizes all experiments conducted on the \textbf{Seq-CIFAR-10} benchmark, where each $32 \times 32$ CIFAR-10 image is flattened into a sequence of $T = 1024$ RGB pixels ($3$-dimensional features), and the model classifies from the last time-step.

\section{Model Architectures}

All three models share the same outer wrapper:
\[
\texttt{Linear}(3 \to D) \;\to\; \tanh \;\to\; \underbrace{[\texttt{Layer}_i + \text{residual}]}_{i=1}^{L} \;\to\; \texttt{LayerNorm} \;\to\; \texttt{Linear}(D \to 10)
\]

\subsection{APN Layer}

Each APN layer maintains a plastic state matrix $S_t \in \mathbb{R}^{D \times D}$ with the gated delta-rule update:
\begin{equation}
S_t = \lambda \, S_{t-1} + \frac{\eta(1-\lambda)}{D} \, \mathbf{k}_t \otimes \bigl(\mathbf{v}_t - S_{t-1}^\top \mathbf{k}_t\bigr)
\end{equation}
where $\mathbf{k}_t = \text{act}(\mathbf{x}_t)$, $\mathbf{v}_t = W \cdot \text{act}(\mathbf{x}_t)$, $\lambda \in (0,1)$ is a trainable decay, and $\eta$ is a trainable learning rate scalar. The output is $\mathbf{o}_t = \mathbf{k}_t^\top S_t + \mathbf{v}_t$ (static path plus dynamic recall).

\textbf{Parameters per layer:} $D^2$ (one linear projection $W$) $+ 2D$ (LayerNorm) $+ 2$ (scalars $\eta, \lambda$) $\approx D^2$.

\subsection{DeltaNet Layer}

DeltaNet uses separate Q/K/V projections with the (non-gated) delta rule:
\begin{equation}
S_t = S_{t-1} + \beta_t \, \mathbf{k}_t \otimes \bigl(\mathbf{v}_t - S_{t-1}^\top \mathbf{k}_t\bigr), \quad \mathbf{o}_t = \mathbf{q}_t^\top S_t
\end{equation}
where $\beta_t = \sigma(W_\beta \mathbf{x}_t)$ is a learned, input-dependent step size.

\textbf{Parameters per layer:} $3D^2$ (Q/K/V projections) $+ D$ ($\beta$ projection) $+ 2D$ (LayerNorm) $\approx 3D^2$.

\subsection{Transformer Layer}

A pre-norm causal Transformer block:
\[
\texttt{LN} \to \texttt{MHA} \to \text{residual} \to \texttt{LN} \to \texttt{FFN}(D \to 4D \to D) \to \text{residual}
\]
Self-attention uses \texttt{F.scaled\_dot\_product\_attention} with \texttt{is\_causal=True}, which auto-dispatches to FlashAttention-2 on CUDA.

\textbf{Parameters per layer:} $3D^2$ (QKV) $+ D^2$ (output proj) $+ 8D^2$ (FFN) $+ 4D$ (two LayerNorms) $\approx 12D^2$.

\subsection{Optional FFN for APN/DeltaNet}

To isolate the effect of the attention mechanism from the FFN, we support an optional \texttt{--use-ffn} flag that adds a pre-norm FFN block (identical to the Transformer's) after each APN or DeltaNet layer. This adds $\approx 8D^2$ parameters per layer.

\textbf{Why APN/DeltaNet can be faster.}
For $T=1024$, $D=100$: the softmax attention FLOPs scale as $T^2 D = 1024^2 \times 100 \approx 10^8$, while the chunked delta-rule scales as $TCD = 1024 \times 64 \times 100 \approx 6.5 \times 10^6$ --- a $\approx 15\times$ reduction in the attention-like computation. However, the APN/DeltaNet kernels are Triton-based (less mature than FlashAttention's hand-tuned CUDA), and at small $D$ the linear projections dominate, so the wall-clock advantage depends on the specific configuration.

For \textbf{inference}, the advantage is even clearer: APN/DeltaNet maintain a fixed $D \times D$ state (constant memory), while the Transformer's KV cache grows linearly with sequence length.

\subsection{Key Observations}

\begin{enumerate}
\item \textbf{APN with \texttt{act=none} is the best APN variant}, reaching 58.17\% (with warmup) versus 51--52\% for \texttt{act=tanh}. The identity activation (no nonlinearity on keys) allows the model to use the full input range without saturating gradients.

\item \textbf{APN is the most parameter-efficient.} With only 103K parameters, APN (act=none) achieves 58.17\%, comparable to DeltaNet at 3$\times$ the parameters (60.80\% with 305K params) and the param-matched transformer at 59.16\%.

\item \textbf{APN is the fastest per epoch.} At $\approx$26s/epoch, APN (act=none, $D=100$) is $\approx$4$\times$ faster than DeltaNet ($\approx$100s) and $\approx$1.5$\times$ faster than the transformer ($\approx$39s) at the same width.

\item \textbf{APN uses the least GPU memory.} 1,590 MB vs 2,133 MB (DeltaNet) vs 4,278 MB (Transformer) at $D=100$. This is expected: APN has $\approx D^2$ params per layer (one projection) versus $3D^2$ (DeltaNet) and $12D^2$ (Transformer).

\item \textbf{Warmup helps slightly.} APN act=none: 57.96\% $\to$ 58.17\% with 20-epoch warmup. The gain is modest but consistent.

\item \textbf{The Transformer is still the most accurate at equal width}, reaching 61.25\% at epoch 65 (still running). This is expected given its $12\times$ parameter advantage. However, even the \emph{parameter-matched} Transformer ($D=29$, 102K params) already hits 59.16\% at epoch 42, suggesting the Transformer architecture is more sample-efficient on this task.
\end{enumerate}

\section{Why APN Is Faster}

Three factors explain APN's speed advantage over DeltaNet and the Transformer:

\begin{enumerate}
\item \textbf{Fewer parameters and projections.}
APN uses a single $D \times D$ projection ($W$) per layer, while DeltaNet uses three ($W_Q, W_K, W_V$) and the Transformer uses four projections plus a $D \to 4D \to D$ FFN.
At $D=100$: APN computes $10$K multiply-adds for projections per layer; DeltaNet computes $30$K; the Transformer computes $120$K.
Fewer projections means fewer GEMM calls, which dominate small-$D$ wall-clock time.

\item \textbf{Linear complexity in $T$.}
The gated delta-rule kernel processes the $T=1024$ sequence in chunks of $C=64$, computing a $C \times C$ triangular system within each chunk and propagating a $D \times D$ state across chunks.
The total work scales as $O(TCD + TD^2/C)$, which is \emph{linear} in $T$.
Softmax attention (even with FlashAttention) is $O(T^2 D)$---quadratic in $T$.
At $T=1024, D=100$: APN's attention FLOPs are $\approx 15\times$ fewer.

\item \textbf{Compact recurrent state.}
APN/DeltaNet store a $D \times D$ matrix per head---$40$~KB at $D=100$ in FP32.
The Transformer must store Q/K/V activations proportional to $T$ for the backward pass, even with FlashAttention's memory-efficient recomputation.
This translates to lower HBM traffic and better cache utilization for the recurrent models.
\end{enumerate}

\textbf{Caveat:} FlashAttention-2 is implemented in hand-tuned CUDA (by Tri Dao), achieving near-peak throughput.
The FLA kernels are written in Triton, which generates good but not optimal GPU code.
As sequence length grows ($T \gg 4096$), the $O(T)$ vs $O(T^2)$ gap widens and the recurrent models' advantage becomes dominant regardless of kernel maturity.

\section{Conclusions}

\begin{enumerate}
\item \textbf{APN achieves competitive accuracy with $\approx$3$\times$ fewer parameters} than DeltaNet and $\approx$12$\times$ fewer than the Transformer, making it the most parameter-efficient model tested.
\item \textbf{APN trains $\approx$4$\times$ faster} than DeltaNet and $\approx$1.5$\times$ faster than the Transformer per epoch at $D=100$, using $\approx$2.7$\times$ less GPU memory than the Transformer.
\item \textbf{The identity activation (\texttt{act=none}) is best for APN}, outperforming \texttt{tanh} by 6+ points.
\item \textbf{The Transformer remains the most accurate} at equal width, but this comes at a $12\times$ parameter cost. The parameter-matched Transformer ($D=29$) is only $\approx$1 point ahead of APN.
\item Upcoming experiments with \textbf{APN + FFN} will test whether adding a feedforward network closes the accuracy gap while retaining APN's speed advantage.
\end{enumerate}
